      // the setup function runs once when you press reset or power the board

        // initialize digital pin LED_BUILTIN,12,11 as an output.

      // the loop function runs over and over again forever
      void loop() {
        digitalWrite(LED_BUILTIN, HIGH);  // turn the green LED on (HIGH is the voltage level)
        delay(500);                       // wait for 0.5 second
        digitalWrite(LED_BUILTIN, LOW);   // turn the green LED off (LOW is the voltage level)
        delay(500);                       // wait for 0.5 second
        digitalWrite(12, HIGH);           // turn the yellow LED on
        delay(500);                       // wait for 0.5 second
        digitalWrite(12, LOW);            // turn the yellow LED off 
        delay(500);                       // wait for 0.5 second

        digitalWrite(11, LOW);            // turn red LED off

      }

This firmware sequentially controls three LEDs (green, yellow, and red), making them blink one after another. Each LED turns on for 0.5 seconds and then off for 0.5 seconds before moving to the next LED. The code is thoroughly commented for clarity, explaining the functionality of each line.

2. Maximum Independent LEDs


The Arduino Uno can blink up to 13 LEDs independently, using all available digital pins. Assuming each LED operates at a desired current of 20mA, the total current draw would be approximately 260mA (0.26A). However, based on research, it is recommended to keep the total current draw within the Arduino Uno's 200mA limit to avoid damaging the board. To achieve this, larger resistors may be required to reduce the current per LED, ensuring the total current stays at or below 200mA. Additionally, the LEDs may need to blink sequentially rather than simultaneously to safely operate within these constraints.


3. Frequency Perception Threshold


Based on testing, the minimum delay time required to perceive the LEDs as blinking is 15 milliseconds for both on and off states. At this speed, the LEDs appear to the human eye as blinking rapidly; any faster (less than 15 milliseconds), they would appear continuously on.
